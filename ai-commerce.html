Conversational Flow Architecture

A typical end-to-end shopping journey powered by an AI agent.

1
User Intent Capture

The shopper expresses intent in natural language ("I need a 50L water heater under $500") — captured by the conversational interface.

2
LLM-Powered Understanding

An LLM parses intent, extracts entities (capacity, price, category), and maps them to SAP Commerce search parameters.

3
OCC API Product Search

Structured query routed to OCC API; SOLR-backed search returns matching products with full PCM data.

4
Conversational Refinement

The agent presents results conversationally and supports follow-up refinements ("Show me only stainless steel models" or "Sort by best rating").

5
Cart Creation via OCC

When the user confirms a selection, the agent calls OCC cart APIs to create or update the cart — all governed by SAP Commerce business rules [1].

6
Guest or Registered Checkout

State-driven flow handles delivery address, payment method, and order placement — supporting both guest and registered user checkout [1].

7
Order Confirmation

OCC returns confirmation; the agent surfaces order details conversationally while SAP Commerce handles fulfillment, ERP sync, and audit trails.

Design Principles

The rules that keep AI-assisted commerce safe, auditable, and enterprise-ready.

🛡️
Separation of AI from Transactional Logic

LLMs handle understanding and language — never direct execution. All commerce operations go through deterministic OCC APIs governed by SAP Commerce [1].

🔒
Enterprise Security & SSO

SAML-based authentication, role-based access, and full audit trails are preserved — no compromises for the sake of conversational UX [1].

📊
Deterministic Pricing & Inventory

Pricing, taxes, promotions, and stock levels are always sourced from SAP Commerce — never inferred or hallucinated by the LLM.

🔄
State-Driven Conversational Flows

Every interaction has explicit states (search, refine, cart, checkout, confirm) — preventing ambiguity and ensuring graceful failure modes [1].

🌐
Channel-Agnostic Design

The same conversational backend can power web chat, mobile chat, voice assistants, or messaging platforms — all driven by OCC.
